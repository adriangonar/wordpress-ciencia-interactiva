El contenido que vamos a publicar

El Problema de los Tres Cuerpos:
de qué trata

Tres objetos en el espacio que se atraen por gravedad. Pregunta: ¿puedes predecir exactamente dónde estará cada uno en el futuro?

Con 2 cuerpos: sí. Kepler lo resolvió en 1609. Fórmula exacta.

Con 3 cuerpos: en general, no existe solución. El sistema es caótico.
Eso sí: podemos simularlo numéricamente con un ordenador. Y esa simulación corre perfectamente en un navegador.
La física — lo justo para entender el código

La visualización principal

¿Dónde puede ir el satélite?
Las Regiones de Hill

Fijada la energía inicial del satélite, hay zonas del espacio que físicamente no puede alcanzar. A esas zonas las llamamos Regiones de Hill.

Cuando la energía sube, se abren "puertas" en los puntos de Lagrange. Primero L1 (paso entre Tierra y Luna), luego L2 (escape al espacio).

Esta imagen —generada píxel a píxel por JavaScript en el navegador— es la visualización central de nuestro artículo WordPress. Y se renderiza en menos de 1 segundo.
Las zonas oscuras son las prohibidas. Los puntos de colores son los Lagrange. Todo calculado en tiempo real.
El núcleo técnico · así está hecho

30 líneas de JavaScript
= física espacial en WordPress

Esta función es el corazón de la simulación. Calcula dónde va el satélite en cada instante. Cabe en cualquier bloque HTML de WordPress.

javascript// Estado del satélite: [x, y, vel_x, vel_y]
function derivadas(x, y, vx, vy, mu) {
  const r1 = Math.sqrt((x+mu)**2 + y**2);
  const r2 = Math.sqrt((x-1+mu)**2 + y**2);

  // Aceleración = suma de fuerzas gravitatorias
  const ax = x + 2*vy
    - (1-mu)*(x+mu)/r1**3
    - mu*(x-1+mu)/r2**3;
  const ay = y - 2*vx
    - (1-mu)*y/r1**3
    - mu*y/r2**3;

  return [vx, vy, ax, ay];
}

¿Cómo entra esto en WordPress?

Bloque HTML personalizado → pegas este código dentro de un <script> junto a un <canvas>. Listo.

¿PHP? No. JavaScript.

WordPress por dentro usa PHP — pero este código no toca el servidor. Corre en el navegador del lector. WordPress es el escaparate. El código científico vive en el frontend.

Sin intermediarios

WordPress no procesa el contenido del bloque HTML. Lo sirve tal cual. El navegador ejecuta el JavaScript directamente. Como conducir un coche sin saber mecánica.

Preguntas sobre WordPress

🧱 ¿Cómo funciona el bloque HTML?

El bloque HTML personalizado sirve el contenido tal cual al navegador del lector. WordPress no lo procesa. El navegador ejecuta el HTML, CSS y JavaScript directamente.

🐘 ¿Necesito PHP para hacer esto?

No. Todo lo que hemos visto es HTML, CSS y JavaScript puro. PHP solo entra si queréis crear un plugin o guardar datos en base de datos. Para publicar simulaciones, no hace falta.

🔒 ¿Es seguro poner JavaScript en el bloque?

El JavaScript corre en el navegador del lector, no en el servidor. No accede a la base de datos ni al backend. El riesgo es idéntico al de cualquier plugin o tema que ya tenéis.

⚡ ¿Afecta al rendimiento?

El script no tiene dependencias externas y usa requestAnimationFrame, que el navegador pausa cuando la pestaña no está visible. El impacto en el tiempo de carga es mínimo.

📱 ¿Funciona en móvil?

Sí. Canvas y requestAnimationFrame son estándar en todos los navegadores móviles modernos. Esta presentación funciona en móvil ahora mismo.

🤔 ¿Por qué no usar un plugin de visualización?

Los plugins de visualización están limitados a gráficas predefinidas — barras, líneas, tartas. El bloque HTML ejecuta cualquier código. No hay comparación en flexibilidad.

Preguntas sobre el contenido

🤔 ¿Por qué WordPress y no una web estática?

WordPress da gestión de contenido, SEO, comentarios, usuarios y API REST sin construir nada desde cero. La simulación vive dentro de ese ecosistema ya construido.

🤔 ¿Por qué este tema científico y no otro?

El Problema de los Tres Cuerpos es visualmente impactante, conecta con misiones reales que el lector conoce (Webb, Artemis), y demuestra el caos de forma inmediata e intuitiva.

🎓 ¿Necesito saber matemáticas para replicar esto?

No. El código es copiable y comentado. Lo que necesitáis es el contexto narrativo — saber qué demuestra cada escenario — que es lo que os lleváis de esta charla.

🌐 ¿Las APIs de la NASA son gratuitas?

Sí. api.nasa.gov tiene APIs públicas y gratuitas: posición de satélites (JPL Horizons), imagen del día (APOD), asteroides cercanos, clima espacial. Se consumen desde JavaScript sin backend.

🔭 ¿Por qué el satélite no escapa en Artemis?

Su energía está entre el nivel de L1 y L2. El cuello de L1 está abierto — puede cruzar hacia la Luna. Pero el de L2 está cerrado — no puede escapar. Es una jaula de energía.

♾️ ¿Qué otros temas funcionarían igual?

Cualquiera con datos o modelos visualizables: epidemias, circuitos, estadística, mapas climáticos de la NASA, ondas sonoras. El patrón es siempre el mismo: modelo + JavaScript + Canvas + bloque HTML.
